

		Создание и удаление таблиц


				Переключиться на другую схему:
				SET search_path TO bookings

				Запрос для создания таблицы:

						CREATE TABLE airports
						(
							airport_code char( 3 ) NOT NULL, -- Код аэропорта
							airport_name text NOT NULL, -- Название аэропорта
							city text NOT NULL, -- гОРОД
							longitude float NOT NULL, -- Координата долготы
							latitude float NOT NULL, -- Координата широты
							timezone text NOT NULL, -- Часовой пояс аэропорта

							PRIMARY KEY ( airport_code )
						);

						Результат: 

					   		 Столбец    |       Тип        | Модификаторы NULL | 
							--------------+------------------+-------------------+
	 						 airport_code | character(3)     | not null          |
	 						 airport_name | text             | not null          |
	 						 city         | text             | not null          |
	 						 longitude    | double precision | not null          |
	 						 latitude     | double precision | not null          |
	 						 timezone     | text             | not null          |

				Посмотреть результат:
				\d airports


				Добавить комментарий к объекам БД:

						COMMENT ON COLUMN airports.city IS 'Город';

				Посмотреть описание столюцов таблицы:
						\d+ airports


		Создадим все таблицы, полный список представлен ниже, представления талиц в файле (tables) :

				1) Таблица Аэропортов (airports)
				2) Таблица Перелетов (flights)
				3) Таблица Бронирования (bookings)
				4) Таблица Билетов (tickets)
				5) Таблица Перелетов (ticket_flights) связана с 
				6) Таблица Посадочных Талонов (boarding_passes)

				Перелеты связаны с Аэропортом отпарвки и прибытия (aircraft_code), а так же с самолетом. 
				Билет связан с Бронированием по ключу book_ref.
				Так как ПЕРЕЛЕТ может состоять из нескольких рейсов, то первичным ключом будет:
				"Номер билета" (ticket_no) и "Идентификатор рейса" (flight_id).
				Посадочные талоны - таблица почти 1:1 - Перелетов, добавляется тольо номер места,
				внешний ключ Таблицы Посадочных талонов ссылатеся на Таблицу Перелетов  

				Так как таблицы связаны, то ри поытке удаления таблицы (airports) возбудится ошибкаю
		Параметры при выводе таблиц:

				"Ссылки из вне" - таблицы которы ССЫЛАЮТСЯ НА ДАННУЮ ТАБЛИЦУ 
				"Ограничения внешнего ключа" - таблицы на которые ССЫЛАЕТСЯ ДАННАЯ таблица
