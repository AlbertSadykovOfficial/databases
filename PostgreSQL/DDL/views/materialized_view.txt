

		Материализованное представление:

				Материализованное представление - предстваление с сохранением данных,
																					таким образом такой вид представлений напоминает
																					отдельную таблицу, но помимо самих данных, он 
																					запоминает еще и запрос с помощью которого
																					данные были собраны.

				Недостаток:

						Необходимо периодически обновлять данные


				Пример:

					 Материализованного представления - routes ("Маршруты")


				Создать:

						CREATE MATERIALIZED VIEW [ IF NOT EXISTS ] <имя-представления>
								[( имя-столбца [, ...])]
								AS <запрос>
								[ WITH [ NO ] DATA ];

						Если выполнить команду с параметром WITH NO DATA, 
						то в момент создания представлени останется пустым,
						для заполнения его данными потребуестя ОБНОВИТЬ данные:

								

				Обновить:

						REFRESH MATERIALIZED VIEW <имя-представления>

				Удалить:

						DELETE MATERIALIZED VIEW <имя-представления>
