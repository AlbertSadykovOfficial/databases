Введение  БД

	Модель данных - совокупноть способов организации БД и досупа к ней
		Виды:
				+ Иерархическая
				+ Сетевая
				+ Реляционная (realation - отношения)

		Отношение - мат. термин из теории множеств.

		Основное ПО БД - database management system (DBMS) (система управления БД - СУБД)

		Процессор языка запросов - ПО, обеспечивающее трансляцию команд высокого уровня (SQL)
															 в запросы.

		Компоненты системы БД:
				+ Клиент - приложения или встроенное ПО СУБД.
				+ Сервер (СУБД)


	Основные поняти реляционной модели

		До появения рел. модели:
				+ БД - набор файлов
				+ Файлы состоят из Записей
				+ Записи состоят из Полей


		В реляционных БД:
				+ БД - набор таблиц (вместо "файл")
				+ Таблицы состоят из Строк (вместо "записей") -> Кортежи
				+ Строки состоят из столбцов (вместо "полей") -> Атрибуты
				+ На пеерсечении стоблцов и строк находятся атомарные значения 

		В теории множеств:
				+ БД - набор таблиц =
				+ Таблицы состоят из Кортежей
				+ Строки состоят из Атрибутов
				Кол-во кортежей(строк) - кардинальное число
				Кол-во арибутов(столбцов) - степень отношения


		Потенциальный и внешний ключ

				Для идентифиации строк в таблицах и для связи таблиц между собой используют КЛЮЧИ.
				
				Потециальный ключ - комбинация атрибутов таблицы, позволяющие идентицфицировать строки в ней.

				Ключ может состоять как из одного, так и из нескольких атрибутов

				При наличие в таблице более 1 потенциального ключа выбрается один - ПЕРВИЧНЫЙ ключ, 
				остальные будут являться альтернативными.


				Внешний ключ - столбец, который связывает данную (ссылающуюся) таблицу
																								с целевой (ссылочной) таблицей 

										   referencing table - ссылающаяся (на) таблица
										   referenced table - ссылочная таблица

		
		Ссылочная целостность

		   	Проблема ссылочной целостноти - проблема обеспечения верных значений внешних ключей.
		   	Ограничение ссылочной целостности - ограничение, согласно которому значения ВНЕШНИХ ключей
		   																			ДОЛЖНЫ СООТВЕСТВОВАТЬ значениям ПОТЕНЦИАЛЬНЫХ ключей.
		   																			Обеспечением выполнения ограничений занимается СУБД.

		   	Каскадное удаление - при удалении/изменении строки из целевой таблицы соответсвующие строки
		   											 из ссылающейся таблицы должны быть удалены/изменены
		   											 (в некоторых случаях особым образом обработаны-> запись NULL значений в ссылки)
		
		
		Транзакции

				Транзакция - набор оперций над БД, рассматриваемых как единая и неделимая единица рабыты
										 (Транзакция либо выполняется польностью, либо не выполняется вовсе)


		SQL

				SQL - язык страктурированных запросов.

				Группы операторов определния данных:
						Data Definition Language  (DDL) - группа операторов определения данных
						Data Manipulation Language(DML) - группа операторов манипулирования данными
						Data Control Language     (DCL) - группа операторов, управляющих привелегиями доступа к объектам БД

				К DDL относятся - команды для создания, изменения, удаления объеков БД
				К DML относятся - команды для выборки строк из таблиц, вставки строк в таблицы, обновления и удлаения строк ()
				К DCL относятся - команды для управления разрешениями на выполнение определенных операция над объеками БД. 

				Примеры:
						DDL: (CREATE, ALTER, DROP)
						DML: (SELECT, INSERT, UPDATE, DELETE)
						DCL: (GRANT, REVOKE, DENY)
