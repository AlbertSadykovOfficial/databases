
		Числовые типы данных


				Целочсленные:
						smallint(int2)
						integer(int4)
						bigint(int8)

				Числа фиксирумой точности:
						numeric(decemical) - могут хранить 131 072 значений до очки и 16 383 после точки. 

						Понятие масштаба и точности:

								Масштаб(scale) - общее число значащих цифр
								Точность(precision) - кол-во цифр, стоящих справа от точки

								Пример:
									12.3456 # Масштаб - 6 Точность - 4

				Числа с плавающей точкой:
						real (кратно 10^37) - точность до 6 десятичных чисел
						double precision (кратно 10^307) - точность до 15 десятичных чисел
						float(n), Если 0<=n<25 - real, если 25<=n<51 - double percision, default -> double percision

						+ Типы поддрживаю специальные значения: Infinity, -Infinity, NaN

						Если значение настолько близко к 0, что для опредееления его значения 
						не хватает точности, будет сгенерирована ошбка Исчезновения знаковых разрядов:
							an underflow error

						! При сравнении данных чисел с плавающей точкой стоит учитывать, что 
							с виду одинаковые числа могут при сравнении выдать неравенство.


				Последовательные типы (serial):
						smallserial
						serial
						bigserial

						Такой тип не является настоящим типом, он используется как удобная замена группы команд.
						Такойтип удобно использовать, когда в какой-лиюо столбец требуется вставить уникальные значения

						Синтаксис создания столбца serial: 

								CREATE TABLE имя-таблицы (имя-столбца serial)

						Аналогично без serial:
								 # nextval - получить следующее значение из последовательности
								 # DEFAULT предписывает значение по умолчанию из nextval

								CREATE SEQUENCE имя-таблицы_имя-столбца_seq
								CREATE TABLE имя-таблицы
								(
										имя-столбца integer NOT NULL
										DEFAULT nextval( 'имя-таблицы_имя-столбца_seq' )
								);
								ALTER SEQUENCE имя-таблицы_имя-столбца_seq
								OWNED BY имя-таблицы.имя-столбца;
