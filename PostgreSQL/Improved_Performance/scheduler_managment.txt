

		Управление планировщиком.

				Для управления планировщиком предусмотрен ряд параметров, которые можно устаавлиать
				параметров SET.

				Изменять параметры планировщика следует аккуратно и при полной уверенности, что
				планировщик ошибается в построении логики выпонения операций.


				Запретить соединение (по умолчанию все значения on):

						Вложенного цикла (nested loop): SET enable_nestloop = off;
						Хэшированием (hash join): SET enable_hashjoin = off;
						Слиянием (merge join): SET enable_mergejoin = off; 

						При этом они не отключаются полостью, методу назначается самая большая
						стоимость выполнения. Это позволяет сохранить возможность маневра.


				При помощи оации ANALIZE можно вывести на экран фактичские затраты времени на выполнение
				запроса и число фактически выбранных строк. (запрос выполняется, но результат не выводится)
				А также выводится доп информация: Где была выполнена сортировка (на диске), затраты памяти
				на сортировку, приведено число строк, удаленных при проверке условия их отбора. 

						EXPLAIN ANALIZE
							...

						Результат:
								...
								Planning time: 122.349 ms
								Execution time: 10941.425 ms

								Фактические затраты - в мс, а оценки стоимости - в условных единицах.
								При этом стоит обратить внимание на число обрабатываемых строк, они 
								оценены достаточно верно.


				При выполнении оепраций с изменением данных (UPDATE...), нужно быть аккуратным, результат
				не выводится, но данные при этом изменются.
				Чтобы проверить выполнение запроса и при этом не изменить данные, следует прибегнуть
				к механизму транзакций, которые позволят сделать откат изменений.

						BEGIN;
						
						EXPLAIN (ANALYZE, COSTS OFF)
						 UPDATE aircrafts
						 	  SET range = range + 100
						 	WHERE model ~ '^Air';

						ROLLBACK;

				!!! НЕЛЬЗЯ экстраполировать оценки, полученные для таблиц небольного размера на таблицы большого размера.
				!!! Это связано с тем, что:
				!!! Оценки, выичисялемые панировщиком не являются линейными.
				!!! К примеру, могут быть выбраны разные планы для проведения запроса.
				!!! Так, для маленькой таблицы может быть выбрано послежовательно сканирование, а для большой - по индексу.
